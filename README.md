# Postgres database performance observability and investigation 
Custom PostgreSQL observability framework for high-throughput systems. It replaces generic exporter metrics with purpose-built SQL views exposed through sql_exporter, scraped by Prometheus, and visualized in Grafana. The system focuses on top resource consumers across queries, tables, indexes, transactions, locks, waits, vacuum activity, and standard database health metrics. Its main value is turning PostgreSQL internals into practical troubleshooting dashboards: engineers can identify heavy SQL, bloat, inefficient indexes, long transactions, hot rows, wait-event bottlenecks, autovacuum lag, and system-wide degradation quickly, both in real time and during post-incident analysis
